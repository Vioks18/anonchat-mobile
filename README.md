# AnonChat Mobile

Анонимный чат-клиент для Android с современным UI, **работающей системой реакций** и продвинутой системой качества кода.

## 🚀 Особенности

- **Анонимное общение** - без регистрации и личных данных
- **Современный UI** - адаптивные темы и анимации
- **🎯 Система реакций** - эмодзи-реакции с красивой анимацией (РАБОТАЕТ!)
- **Продвинутая QA система** - автоматическая проверка качества кода
- **Pre-commit защита** - блокировка критических ошибок
- **TypeScript** - полная типизация
- **React Native + Expo** - кроссплатформенная разработка

## 🛠️ Технологии

- **React Native** - мобильная разработка
- **Expo** - инструменты разработки
- **TypeScript** - типизация
- **Zustand** - управление состоянием
- **React Native Gesture Handler** - жесты
- **React Native Portalize** - порталы для UI
- **Expo Vector Icons** - иконки

## 🛡️ QA система (v2.0)

### Быстрый старт
```bash
# Проверка качества кода
npm run qa

# Автоисправление критических проблем
npm run qa:fix

# Детальная проверка с отладкой
npm run qa:debug
```

### Режимы работы
- **strict** (по умолчанию) - исключает experimental правила
- **soft** - полная проверка всех правил

### Команды QA
```bash
npm run qa              # Обычная проверка (strict)
npm run qa:strict       # Строгий режим
npm run qa:soft         # Мягкий режим (все правила)
npm run qa:debug        # С детальным выводом
npm run qa:fix          # Автоисправление P0 проблем
npm run qa:baseline:update  # Обновить baseline
```

### LOC инструменты и защита от больших файлов
```bash
npm run stats:loc       # Подсчет строк кода в app/
```

**Защита от больших файлов:**
- Максимальный размер файла: 300 строк (по умолчанию)
- Исключения: `app/components/ChatCore.styles.ts`
- Автоматическая проверка в QA системе
- Исключает backup, mocks, node_modules, .expo, android, ios, dist, build

**Отчеты:**
- `qa/loc-report.json` - детальная статистика по файлам
- Выводит топ-20 файлов по размеру
- Интегрирован в CI/CD pipeline

### Corridor Guard система
```bash
npm run qa:corridor     # Проверка corridor режима
```

**Назначение:**
- Предотвращает случайные изменения вне разрешенных файлов во время corridor режима
- Обеспечивает фокус на конкретных файлах при рефакторинге

**Как использовать:**
1. Добавьте файлы в `qa-allowlist.json` в массив `"allowed"`
2. Запустите `npm run qa:corridor` для проверки
3. Система автоматически разрешает: `package.json`, `docs/**`, `scripts/qa/**`

**Pre-commit защита:**
- Максимум 1 файл в allowlist во время corridor режима
- Автоматическая проверка при коммите
- Четкие сообщения об ошибках

### Pre-commit защита
- Автоматический запуск QA при коммите
- Блокировка при наличии P0 ошибок
- Защита от console.log в продакшене

## 🎯 Система реакций - РАБОТАЕТ!

### Функциональность:
- ✅ **Long press** на сообщение → выбор
- ✅ **Double tap** на сообщение → реакция
- ✅ **Tap на эмодзи** в панели → добавление/удаление
- ✅ **Tap на чип реакции** → toggle реакция
- ✅ **Скролл** → закрытие панели реакций
- ✅ **Удаление сообщения** → закрытие меню реакций
- ✅ **Удаление для всех** → скрытие реакций

### UI особенности:
- Плавающая панель с анимацией
- Автоматическое позиционирование
- Адаптивные пузыри сообщений
- Компактные чипы реакций

### Производительность:
- React.memo на компонентах
- useCallback для обработчиков
- Мемоизация селекторов
- Оптимизированные ререндеры

## 🔧 Разработка

### Правила кода
1. Все console.log должны быть в `if (__DEV__)`
2. Используйте TypeScript для типизации
3. Следуйте принципам функционального программирования
4. Мемоизируйте компоненты и обработчики

### Git workflow
```bash
# Создание новой фичи
npm run prefeature

# Проверка перед коммитом (автоматически)
git commit -m "feat: new feature"

# Создание бэкапа
npm run backup
```

### QA workflow
```bash
# Ежедневная проверка
npm run qa

# При проблемах
npm run qa:debug

# Автоисправление
npm run qa:fix

# Обновление baseline
npm run qa:baseline:update
```
