# 🔄 Система автоматических бэкапов AnonChat

## 📁 Структура бэкапов

```
backup/
├── components/
│   ├── ChatCore.tsx.backup
│   ├── UIErrorBoundary.tsx.backup
│   └── ...
├── hooks/
│   ├── useMessageStore.ts.backup
│   └── ...
├── screens/
│   ├── ChatScreen.tsx.backup
│   └── ...
├── context/
│   ├── ChatLogicProvider.tsx.backup
│   └── ...
└── README.md
```

## 🎯 Назначение

- **Быстрое восстановление** при авариях
- **Сохранение актуальных фич** (темы, меню, Zustand)
- **История изменений** для отладки
- **Безопасность** при экспериментах

## 📋 Критически важные файлы для бэкапа

### ✅ Обязательные
- `app/components/ChatCore.tsx` - UI Fortress
- `app/hooks/useMessageStore.ts` - Zustand Store
- `app/screens/ChatScreen.tsx` - Главный экран
- `app/index.tsx` - Точка входа

### 🔧 Дополнительные
- `app/components/UIErrorBoundary.tsx` - Обработка ошибок
- `app/context/ChatLogicProvider.tsx` - Контекст
- `app.config.js` - Конфигурация Expo

## 🚀 Команды для бэкапа

```bash
# Создать бэкап всех критических файлов
npm run backup

# Восстановить из бэкапа
npm run restore

# Обновить конкретный файл
npm run backup:core
```

## 📝 Правила бэкапов

1. **Создавать бэкап** после каждой успешной фичи
2. **Тестировать** перед созданием бэкапа
3. **Комментировать** что добавлено в бэкапе
4. **Не удалять** старые бэкапы без необходимости

## 🔄 Когда создавать бэкап

- ✅ После добавления новой темы
- ✅ После интеграции Zustand Store
- ✅ После исправления UI багов
- ✅ После добавления новых функций
- ✅ Перед экспериментами

## 🛠️ Восстановление

```bash
# Полное восстановление
npm run restore:full

# Восстановление конкретного файла
npm run restore:core

# Восстановление из определенной даты
npm run restore:date 2024-01-15
```
