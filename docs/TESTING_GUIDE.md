# Руководство по тестированию аутентификации

## 🧪 Тестирование регистрации

### 1. Успешная регистрация
```
1. Открыть приложение
2. Нажать "Создать аккаунт"
3. Ввести валидные данные:
   - Username: testuser123
   - Email: test@example.com
   - Password: password123
4. Нажать "Создать аккаунт"
5. Ожидать: переход в чат
```

### 2. Проверка уникальности username
```
1. Зарегистрировать пользователя с username "testuser"
2. Выйти из аккаунта
3. Попробовать зарегистрировать второго с тем же username
4. Ожидать: ошибка "Этот username уже занят"
```

### 3. Валидация username
```
Тест 1: Короткий username
- Ввести: "ab"
- Ожидать: ошибка "минимум 3 символа"

Тест 2: Длинный username
- Ввести: "verylongusername123456789"
- Ожидать: ошибка "максимум 20 символов"

Тест 3: Недопустимые символы
- Ввести: "user@name"
- Ожидать: ошибка "только буквы, цифры и подчеркивания"

Тест 4: Зарезервированные слова
- Ввести: "admin"
- Ожидать: ошибка "зарезервирован"

Тест 5: Гибридная система @
- Ввести: "testuser" → должно работать
- Ввести: "@testuser" → должно работать
- Ввести: "TestUser" → должно работать (нормализация)
```

## 🔐 Тестирование входа

### 1. Успешный вход
```
1. Зарегистрировать пользователя
2. Выйти из аккаунта
3. Войти с правильными данными
4. Ожидать: переход в чат с загруженным username
```

### 2. Неправильные данные
```
1. Попробовать войти с неправильным email
2. Ожидать: ошибка аутентификации

1. Попробовать войти с неправильным паролем
2. Ожидать: ошибка аутентификации
```

## 🔄 Тестирование состояния

### 1. Автоматический вход
```
1. Зарегистрироваться и войти
2. Закрыть приложение
3. Открыть приложение снова
4. Ожидать: автоматический вход без ввода данных
```

### 2. Выход из аккаунта
```
1. Войти в аккаунт
2. Выйти из аккаунта
3. Ожидать: возврат на экран входа
```

## 🗄️ Проверка Firestore

### 1. Коллекция usernames
```
1. Зарегистрировать пользователя с username "testuser"
2. Открыть Firebase Console
3. Перейти в Firestore
4. Проверить коллекцию usernames
5. Ожидать: документ testuser с uid пользователя
```

### 2. Коллекция users
```
1. Зарегистрировать пользователя
2. Проверить коллекцию users
3. Ожидать: документ с uid пользователя и полным профилем
```

## ⚠️ Тестирование ошибок

### 1. Сетевые ошибки
```
1. Отключить интернет
2. Попробовать зарегистрироваться
3. Ожидать: ошибка сети

1. Включить интернет
2. Попробовать снова
3. Ожидать: успешная регистрация
```

### 2. Race conditions
```
1. Открыть два устройства/браузера
2. Одновременно попытаться зарегистрироваться с одним username
3. Ожидать: один успех, один провал
```

## 📱 Тестирование UI

### 1. Responsive дизайн
```
1. Тестировать на разных размерах экрана
2. Проверить адаптивность форм
3. Убедиться в читаемости текста
```

### 2. Accessibility
```
1. Проверить работу с screen reader
2. Тестировать навигацию с клавиатуры
3. Убедиться в контрастности цветов
```

## 🚀 Производительность

### 1. Время загрузки
```
1. Измерить время регистрации
2. Измерить время входа
3. Ожидать: < 3 секунд для каждой операции
```

### 2. Кэширование
```
1. Войти в аккаунт
2. Перезапустить приложение
3. Ожидать: быстрый вход без повторной загрузки данных
```

## 📊 Метрики успеха

- ✅ Регистрация работает без ошибок
- ✅ Username уникальность соблюдается
- ✅ Валидация работает корректно
- ✅ Автоматический вход функционирует
- ✅ Данные сохраняются в Firestore
- ✅ UI отзывчивый и понятный
- ✅ Ошибки обрабатываются gracefully
