# Система Уникальных Имен Пользователей

## Обзор

Система уникальных имен пользователей позволяет пользователям выбирать понятные имена вместо длинных UID, сохраняя при этом анонимность и безопасность.

## Архитектура

### База данных

#### Коллекция `users/{uid}`
```typescript
{
  username: string,      // "@Alex" - отображаемое имя с @
  username_lc: string,   // "alex" - нижний регистр для уникальности
  // ... другие поля пользователя
}
```

#### Коллекция `usernames/{handle}`
```typescript
{
  uid: string,           // UID владельца
  display: string,       // "Alex" - отображаемое имя без @
  createdAt: timestamp   // Время создания
}
```

### Принципы работы

1. **Уникальность**: Документ в `usernames/{handle}` создается только один раз
2. **Case-insensitive**: `@Alex` и `@alex` считаются одинаковыми
3. **Атомарность**: Используются Firestore транзакции
4. **Безопасность**: Клиенты могут только создавать, не обновлять/удалять

## Валидация

### Правила валидации

- **Длина**: 3-20 символов
- **Символы**: `a-z`, `A-Z`, `0-9`, `_`
- **Начало**: Не может начинаться с подчеркивания
- **Регистр**: Автоматически приводится к нижнему для уникальности

### Зарезервированные имена

```typescript
const RESERVED_USERNAMES = [
  'admin', 'support', 'help', 'moderator', 'system', 'anon', 
  'me', 'you', 'null', 'root', 'owner'
];
```

### Нормализация

```typescript
// Вход: "@John_Doe123"
// ID: "john_doe123" (для уникальности)
// Display: "John_Doe123" (для отображения)
```

## API

### Основные функции

#### `validateHandle(input: string)`
Проверяет корректность имени пользователя.

#### `checkUsernameAvailable(handleInput: string)`
Проверяет доступность имени.

#### `claimUsername(handleInput: string, uid: string)`
Заявляет имя для пользователя (атомарная операция).

#### `suggestRandomHandle(uid: string)`
Генерирует случайное имя на основе UID.

### Примеры использования

```typescript
// Проверка доступности
const availability = await checkUsernameAvailable('john_doe');
if (availability.available) {
  console.log('Имя доступно!');
}

// Заявление имени
const result = await claimUsername('john_doe', userId);
if (result.success) {
  console.log(`Имя @${result.display} установлено!`);
}
```

## Безопасность

### Firestore Rules

```javascript
// Пользователи могут читать профили
match /users/{userId} {
  allow read: if isSignedIn();
  allow write: if isSignedIn() && request.auth.uid == userId;
}

// Имена - только создание, без обновления/удаления
match /usernames/{handle} {
  allow read: if true; // Публичное чтение
  allow create: if isSignedIn() && 
    !exists(/databases/$(database)/documents/usernames/$(handle)) && 
    request.resource.data.uid == request.auth.uid;
  allow update, delete: if false;
}
```

### Транзакции

Все операции с именами выполняются в транзакциях для гарантии атомарности:

1. Проверка существования имени
2. Создание документа в `usernames/{handle}`
3. Обновление документа в `users/{uid}`

## UI/UX

### UsernameScreen

- **Валидация в реальном времени** с debounce 400ms
- **Проверка доступности** с визуальной обратной связью
- **Предложение случайного имени** кнопкой
- **Правила валидации** отображаются внизу экрана

### Навигация

- **Гейт**: Показывается только если у пользователя нет имени
- **Автоматический переход**: После установки имени → ChatListScreen
- **Fallback**: Если ошибка → остается на экране с сообщением

```json

  "fields": [
    {"fieldPath": "username_lc", "order": "ASCENDING"},
```

### Кэширование

- **Debounce**: 400ms для проверки доступности
- **Оптимизированные запросы**: Минимальное количество обращений к Firestore
- **Локальная валидация**: Проверка на клиенте перед запросом

## Миграция

### Совместимость

- **Обратная совместимость**: Существующие пользователи без имен продолжают работать
- **Постепенная миграция**: Пользователи устанавливают имена при первом входе
- **Fallback**: Если имя не установлено, показывается UID

### Автоматическая установка

```typescript
// Генерация временного имени
const generateTempUsername = (uid: string): string => {
  return `anon_${uid.slice(0, 4)}${Math.floor(Math.random() * 900) + 100}`;
};
```

## Ограничения

### MVP ограничения

- **Нет переименования**: Имя устанавливается один раз
- **Нет удаления**: Имена нельзя удалить
- **Нет передачи**: Имена нельзя передать другому пользователю

### Технические ограничения

- **Case-insensitive**: `@Alex` и `@alex` одинаковы
- **Длина**: Максимум 20 символов
- **Символы**: Только буквы, цифры, подчеркивания
- **Зарезервированные**: Список заблокированных имен
